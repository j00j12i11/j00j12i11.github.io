# 정보처리기사 실기 시험 복기 도우미

## 목적

이 프로젝트는 **정보처리기사 실기 시험** 준비를 위한 기출 복기 및 개념 정리 공간이다.
사용자는 기출 문제를 푼 뒤, 1번부터 한 문제씩 대화하며 복기한다.

이 저장소는 **옵시디언(Obsidian) Vault**로 관리된다. 파일 작성 시 옵시디언 호환성(위키링크 `[[ ]]`, 태그 `#태그`, frontmatter, Dataview 쿼리)을 고려한다.

## 폴더 구조

```
개념노트/       주제별 단일 파일 (암기/훑어보기용)
기출복기/       회차별 복기 기록 (문제별 정오답 + 관련 개념 링크)
규칙/           세부 규칙·기준 파일
  ├─ 목차.md        정처기 표준 목차 (배치 기준)
  ├─ 개념노트.md    개념노트 저장·정제 규칙
  ├─ 기출복기.md    기출복기 저장 규칙 + 회차 종료 시 정리 절차
  └─ 오답문풀.md    오답 재풀이 진행 규칙
```

## 규칙 파일 로드 시점

- **개념노트 작성·갱신·정리** → `규칙/개념노트.md` 로드 후 진행.
- **기출복기 작성·회차 종료 정리** → `규칙/기출복기.md` 로드 (개념노트 규칙도 함께 참조).
- **오답문풀** → `규칙/오답문풀.md` 로드 (Sonnet 모델 권장).
- **새 개념이 어느 파일·섹션에 들어갈지 불확실** → `규칙/목차.md` 확인.

## 랜덤퀴즈

사용자가 "랜덤퀴즈", "퀴즈" 등을 요청하면:

1. 모델 확인 — Sonnet 권장.
2. 출제 방식 — 10문제 미리 생성 후 1문제씩 보여줌.
3. 형식 — 단답형·빈칸뚫기 위주. 코드 트레이싱 출제 X.
4. 출제 소스 — `개념노트/` 파일들의 내용 기반.

## 대량 편집 가드레일

옵시디언 볼트는 위키링크·메타블록·반복 헤딩 등 동일 패턴이 많아 대량
치환 작업이 잦다. 동일 파일을 단순 Edit으로 반복하면 프롬프트 캐시가
매번 무효화되어 토큰이 수십 배로 불어난다.

| 트리거 | 전환 |
|---|---|
| 동일 파일 Edit 5회 초과 | Write 전체 재작성 검토 |
| 동일 패턴 치환 10회 초과 | `Edit replace_all` / `MultiEdit` / `Bash + sed` |
| 세션 누적 Edit 50회 초과 | 즉시 중단, 일괄 처리 재설계 |

위키링크 일괄 치환은 `sed`/`perl -pi` 한 번으로 끝낸다. 단순 Edit
반복으로 풀지 않는다.
