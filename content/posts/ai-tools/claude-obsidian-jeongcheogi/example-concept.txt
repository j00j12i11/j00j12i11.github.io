# 개념노트 저장 규칙

## 파일 구성

- 복기한 문제에서 다룬 개념을 `개념노트/` 폴더에 정리한다.
- 파일은 [[목차]]의 대분류 기준으로 구성한다.
- 현재 파일 구성:
  - `소프트웨어구축.md` — 목차 1번 (요구사항·설계·구현·테스트 등)
  - `데이터베이스.md` — 목차 2번
  - `운영체제.md` — 목차 3번
  - `네트워크.md` — 목차 4번
  - `정보보안.md` — 목차 5번
  - `기타.md` — 목차 6번
  - `SQL.md` — 목차 7번
  - `C언어.md`, `Java.md`, `Python.md` — 프로그래밍 언어별 별도

## H2 네이밍 규칙

- H2 헤딩은 **괄호·em dash·부가설명 없이 간결하게** 작성한다.
  - O: `## BCP`, `## 반정규화`, `## CIA`
  - X: `## BCP (RTO · RPO)`, `## 반정규화(Denormalization)`

## 메타블록 (항목별 필수)

각 개념 헤딩 바로 아래에 Dataview 인라인 필드로 메타블록을 둔다.

```markdown
## 자료형
- [출제:: 7] [오답:: 1] [최근:: 2025-3회]
- 이력
  - [[2025-3회]] 11번 ✓
  - [[2022-1회]] 8번 ✓
  - [[2020-2회]] 2번 ✗
```

(이하 생략 — 위키링크 연결 레벨, 정제 원칙, 프로그래밍 분량 차등 등)
